Proyectos:
Taller mecánico (Web): Simulación de un taller mecánico utilizando PHP y MySQL. Implementamos el patrón de diseño MVC (Modelo-Vista-Controlador) para organizar el código, separando la lógica de la aplicación en tres componentes interconectados. Esto nos permitió mejorar la mantenibilidad del código y facilitar el trabajo en equipo, ya que cada miembro podía enfocarse en un componente específico.

Modelo: Gestionaba la información de clientes, vehículos, reparaciones y stock de repuestos, interactuando con la base de datos MySQL.
Vista: Se encargaba de la presentación de la información al usuario, utilizando HTML y CSS para crear una interfaz intuitiva.
Controlador: Gestionaba las peticiones del usuario (registro de clientes, creación de órdenes de trabajo, etc.) y la lógica de la aplicación (cálculo de costos, generación de informes).

Sistema de pedidos para minimarket (Móvil): Aplicación Android desarrollada en Kotlin con Android Studio que simula un sistema de pedidos para un minimarket.
Utilizamos el patrón de diseño MVVM (Modelo-Vista-ViewModel) para crear una aplicación robusta y fácil de mantener.

Modelo: Representaba los datos de productos, clientes y pedidos.
Vista: Mostraba la interfaz de usuario para que los clientes pudieran navegar por los productos, agregarlos al carrito y realizar pedidos.
ViewModel: Actuaba como intermediario entre la vista y el modelo, proporcionando los datos necesarios para la vista y gestionando la lógica de la aplicación (validación de datos, procesamiento de pedidos).

Videojuego educativo (Unity): Proyecto de videojuego para mejorar la comprensión lectora de niños de educación básica. Se aplicó el método del silabario hispanoamericano y se utilizó el motor gráfico Unity con C#. En este proyecto, consideramos principios de Clean Architecture para asegurar la escalabilidad y la testabilidad del código.

Entidades: Representan los conceptos básicos del juego, como el jugador, los enemigos y los objetos del escenario.
Casos de uso: Implementan la lógica del juego, como el movimiento del jugador, la interacción con objetos y la resolución de puzzles.
Adaptadores: Se encargan de la interacción con frameworks externos, como la interfaz gráfica de usuario y el motor de física.
